# Universal Validation Checklist
## Apply to ALL Phases and ALL Findings

**Version:** 1.0
**Date:** 2025-09-14
**Purpose:** Standardized validation across all analysis phases

---

## üéØ MANDATORY FOR EVERY FINDING

### Level 1: Basic Validation (ALL findings)
- [ ] **Specific not generic** - Named technologies, not categories
- [ ] **Evidence provided** - At least one source cited
- [ ] **China angle explicit** - How China could exploit
- [ ] **Impact quantified** - Numbers, not "significant"

### Level 2: Standard Validation (Important findings)
- [ ] **Multiple sources** - 2+ independent sources
- [ ] **Timeline verified** - Dates and sequence confirmed
- [ ] **Actors identified** - Named entities, not "Chinese interests"
- [ ] **Pathway documented** - How exploitation would occur
- [ ] **Alternatives considered** - At least 2 other explanations

### Level 3: Deep Validation (Critical/extraordinary findings)
- [ ] **Triple source rule** - 3+ independent sources
- [ ] **Primary documents** - Official records reviewed
- [ ] **Archive captured** - Critical sources archived
- [ ] **5+ alternatives tested** - Full hypothesis testing
- [ ] **Confidence scored** - 0-20 scale calculated
- [ ] **Gap analysis complete** - How this happened explained
- [ ] **Contradiction search** - Negative evidence sought
- [ ] **Peer review** - Second analyst review if available

## üìä TECHNOLOGY ASSESSMENT CHECKLIST

### For EVERY Technology Mentioned:

#### Maturity Assessment
- [ ] **TRL assigned** (1-9 scale)
  - TRL 1-3: Basic research
  - TRL 4-6: Development
  - TRL 7-8: Demonstration
  - TRL 9: Operational

#### Strategic Value
- [ ] **China advancement calculated** (years saved)
  - CRITICAL: 5-10 years leapfrog
  - HIGH: 2-5 years acceleration
  - MEDIUM: 1-2 years benefit
  - LOW: <1 year advantage
  - NONE: No strategic value

#### Overlap Analysis
- [ ] **US-China overlap identified**
  - Exact same product/system
  - Shared platform/base
  - Technology family
  - No overlap

#### Access Assessment
- [ ] **China access level determined**
  - Physical possession
  - Documentation access
  - Observation only
  - No access

## üîç EVIDENCE QUALITY MATRIX

### Source Hierarchy (best to worst):
1. **Primary Documents**
   - [ ] Government filings
   - [ ] Corporate reports
   - [ ] Official contracts
   - [ ] Patent filings

2. **Authoritative Secondary**
   - [ ] Industry analysts
   - [ ] Academic papers
   - [ ] Trade publications
   - [ ] Expert interviews

3. **Credible Tertiary**
   - [ ] Major news outlets
   - [ ] Think tank reports
   - [ ] Conference proceedings
   - [ ] Industry newsletters

4. **Weak Sources** (avoid)
   - [ ] Blogs/opinion
   - [ ] Social media
   - [ ] Anonymous claims
   - [ ] Outdated (>5 years)

### Corroboration Requirements:
- **Routine claim**: 1 credible source
- **Important claim**: 2+ diverse sources
- **Critical claim**: 3+ independent sources
- **Extraordinary claim**: 3+ sources + archives + alternatives tested

## üéØ ALTERNATIVE HYPOTHESIS CHECKLIST

### Must Generate & Test (minimum 5):

#### 1. Business Logic
- [ ] Normal profit motive?
- [ ] Market competition?
- [ ] Customer demand?
- [ ] Standard practice?

#### 2. Regulatory
- [ ] Legal requirement?
- [ ] Compliance driven?
- [ ] Tax optimization?
- [ ] Trade agreement?

#### 3. Historical
- [ ] Pre-existing relationship?
- [ ] Legacy arrangement?
- [ ] Different era decision?
- [ ] Being phased out?

#### 4. Technical
- [ ] Different products?
- [ ] Various configurations?
- [ ] Generation gaps?
- [ ] Capabilities differ?

#### 5. Misinterpretation
- [ ] Translation error?
- [ ] Context missing?
- [ ] Assumption wrong?
- [ ] Data mistake?

### Testing Protocol:
1. List all hypotheses
2. Assign prior probabilities
3. Identify discriminating evidence
4. Collect evidence
5. Update probabilities
6. Document conclusion

## üìã OVERSIGHT GAP IDENTIFICATION

### For Every Vulnerability Found:

#### Gap Type Assessment
- [ ] **Organizational silo** - Agencies not talking
- [ ] **Temporal discontinuity** - Old decision, new reality
- [ ] **Classification paradox** - Can't connect dots
- [ ] **Regulatory arbitrage** - Exploiting rule differences
- [ ] **Incentive misalignment** - Everyone wins but security

#### Gap Analysis Requirements
- [ ] **How formed** - Historical evolution
- [ ] **Why persists** - Current incentives
- [ ] **Who benefits** - Stakeholder mapping
- [ ] **Fix difficulty** - Barriers to closure
- [ ] **Exploitation risk** - China awareness level

## üö® BOMBSHELL SCORING (When Finding Seems Extraordinary)

### Calculate Score (1-5 each, 30 max):

| Factor | Score | Description |
|--------|-------|-------------|
| **Sameness** | ___ | How identical is technology? |
| **Impact** | ___ | Damage to US interests? |
| **Intent** | ___ | Deliberate or accidental? |
| **Awareness** | ___ | Who knows about this? |
| **Alternatives** | ___ | Other explanations viable? |
| **Evidence** | ___ | How solid is proof? |
| **TOTAL** | ___/30 | |

### Interpretation:
- 25-30: DEFINITE BOMBSHELL - Escalate immediately
- 20-24: PROBABLE BOMBSHELL - Investigate further
- 15-19: SIGNIFICANT FINDING - Document carefully
- 10-14: NOTEWORTHY - Include in analysis
- 5-9: ROUTINE - Standard risk

### Before Declaring Bombshell:
- [ ] Score calculated >20
- [ ] Alternatives inadequate to explain
- [ ] Evidence triple verified
- [ ] Archive captures complete
- [ ] Peer review if possible
- [ ] Language measured (not alarmist)
