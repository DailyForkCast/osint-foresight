## üîë Key Features of v6.0

### Core Frameworks
| Framework | Description | Threshold/Requirement |
|-----------|-------------|----------------------|
| **Leonardo Standard** | 8-point technology specificity checklist | All tech mentions |
| **Bombshell Validation** | 6-factor scoring system | Score >20 triggers |
| **Conference Intelligence** | 2020-2030 event tracking | Tier 1: 3+ countries with China |
| **Arctic Policy** | Refined focus on primary Arctic states | Survey others first |
| **Failsafe Protocol** | Include critical findings even with gaps | Value weight: Critical=10 |

### Evidence Standards
- Roman numeral citations: (i), (ii), (iii)
- Source tags: [WEB], [GOV], [CORP], [ACAD], [CC], [REG]
- Archive requirements for critical claims
- SHA-256 hashing for sensitive evidence

### Narrative Requirements
- 600-1,500 words per phase
- "What It Means" sections mandatory
- Topic sentences and transitions required
- Probability bands: [10,30), [30,60), [60,90)

## ‚ö° Common Tasks

### Starting New Country Analysis
1. Check if country is in PRIORITY_COUNTRIES
2. If primary Arctic state ‚Üí Include full Arctic analysis
3. If not primary Arctic ‚Üí Apply Arctic survey first
4. Use appropriate master prompt (ChatGPT or Claude Code)
5. Follow phase progression 0-13

### Validating Extraordinary Claims
1. Apply Bombshell Validation (6 factors)
2. Score each factor 1-5
3. If total >20 ‚Üí Investigate further
4. If total >25 ‚Üí Escalate immediately

### Conference Assessment
1. Check if event has China + Target + US
2. Count participating countries
3. For Arctic conferences ‚Üí Only Tier 1 for primary Arctic states
4. Apply classification framework
